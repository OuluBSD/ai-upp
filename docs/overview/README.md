# Overview Docs

`docs/overview/` contains architectural and semantic guides for major repository areas. These files are meant to answer "what is this layer for, what semantics does it impose, and where do platform or build differences matter?" They are not full API references.

Conventions:

- topic collections live in subdirectories such as [`Core/`](Core/README.md)
- ordered topic files use two-digit prefixes like `00-`, `01-`, `02-`
- top-level indexes live in [`INDEX.md`](INDEX.md) and [`HIERARCHY.md`](HIERARCHY.md)

Current sections:

- [Core/README.md](Core/README.md): foundational runtime layer overview and topic index.
