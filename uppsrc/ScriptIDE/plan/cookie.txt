## Critical Constraints

### ScriptCommon Boundary Policy
- `ScriptCommon` and `ScriptCLI` must not depend on `CtrlCore` or `CtrlLib`.
- Non-GUI runtime/services move to `ScriptCommon`.
- GUI wrappers remain in `ScriptIDE`.
- Pane logic should be extracted to headless services so CLI/MCP/TUI can reuse it.

### Migration Reminder
When old plan tasks mention these files in `ScriptIDE`, interpret with split intent:
- `RunManager`, `Linter`, `PathManager`, `IDESettings` -> `ScriptCommon`
- Plugin core contracts/registry -> `ScriptCommon`
- UI controls/panes/windows -> `ScriptIDE` adapters/views
